1111.tn API Reference

All endpoints consumed by the Flutter mobile app. All responses are JSON.

Base URL https://production.1111.tn
Overview
Two base URLs: The backend (auth, users) runs on the gateway at port 3000. The product/stats APIs are served by the Next.js frontend.
ServiceBase URLAuth required
Auth & Users (gateway)https://production.1111.tn:3000Some endpoints
Products, Stats, Search (Next.js)https://production.1111.tnNo
Authorization header (authenticated endpoints)
// Add to every request that requires login:
Authorization: Bearer <access_token>
Error response format (gateway)
{
  "success": false,
  "message": "Error description"
}
Error response format (Next.js APIs)
{
  "error": "Error message string"
}
Pagination (product list endpoints)
// List endpoints support these common query params:
page   // integer, 0-based offset (default: 0)
limit  // integer, items per page (default: 24, max: 48)

// Response always includes:
{
  "total": 1234,
  "page":  0,
  "limit": 24,
  "items": [ ... ]

Authentication
Base URL: https://production.1111.tn:3000/api/auth

Public endpoints (no token needed): register, login, google, refresh, forgot-password, reset-password
Protected (require Authorization: Bearer <access_token>): verify-email, resend-otp
POST /auth/register Create a new user account

Registers a new user. Sends a welcome email + 6-digit OTP verification code. Returns tokens immediately so the user can use the app before verifying.

Request Body (JSON)
FieldTypeRules
full_name requiredstring2–100 chars
email optional*stringValid email. Either email or phone required.
phone optional*stringTunisian format: +216XXXXXXXX
password requiredstring8–128 chars
state requiredstringOne of 24 Tunisian governorates (e.g. Tunis, Sfax, Sousse…)
Response 201
{
  "success": true,
  "message": "Registration successful",
  "data": {
    "user": {
      "id":                "uuid",
      "full_name":         "Wala Naghmouchiwala",
      "email":             "wala@example.com",
      "phone":             null,
      "google_id":         null,
      "avatar_url":        null,
      "role":              "user",
      "state":             "Tunis",
      "is_email_verified": false,
      "is_active":         true
    },
    "access_token":  "eyJhbGci...",   // JWT, short-lived (~15 min)
    "refresh_token": "selector.secret", // long-lived, store securely
    "redirect_to":   "front-office"    // "front-office" | "back-office"
  }
}
Error codes
StatusMessage
409Email already in use / Phone already in use
400Validation error (missing fields, bad format)

Request Body (JSON)
FieldTypeRules

Response 200

POST /auth/google Login / register with Google OAuth

Use Google Sign-In SDK on mobile to get an id_token, then send it here. Creates the account if it doesn't exist yet.

Request Body (JSON)
FieldTypeRules
id_token requiredstringGoogle ID token from the OAuth flow
state optionalstringTunisian governorate (required on first registration)
Response 200
{
  "success": true,
  "data": {
    "user":          { /* user object, is_email_verified: true */ },
    "access_token":  "eyJhbGci...",
    "refresh_token": "selector.secret",
    "redirect_to":   "front-office"
  }
}
Error codes
StatusMessage
400Invalid Google token
403Google email must be verified / Account disabled
503Google OAuth not configured
POST /auth/refresh Rotate tokens using refresh token

Exchanges a valid refresh token for a new access + refresh token pair. The old refresh token is immediately revoked (rotation). Call this when the access token expires.

Request Body (JSON)
FieldTypeRules
refresh_token requiredstringThe refresh token from login/register
Response 200
{
  "success": true,
  "data": {
    "access_token":  "eyJhbGci...",   // new access token
    "refresh_token": "selector.secret"  // new refresh token — save this, old one is dead
  }
}
Error codes
StatusMessage
401Invalid or expired refresh token
POST /auth/logout Revoke refresh token (logout)
Request Body (JSON)
FieldTypeRules
refresh_token requiredstringThe refresh token to revoke
Response 200
{
  "success": true,
  "message": "Logged out"
}

Always returns 200 even if the token was already revoked or invalid — no information is leaked.

POST /auth/verify-email Verify email with 6-digit OTP

Requires a valid Authorization: Bearer <access_token> header. The OTP is emailed after registration.

Headers
HeaderValue
AuthorizationBearer <access_token>
Request Body (JSON)
FieldTypeRules
otp requiredstringExactly 6 digits
Response 200
{ "success": true, "message": "Email verified" }
Error codes
StatusMessage
400OTP not found or expired / Invalid OTP / OTP invalidated — request a new one
401Unauthorized (missing token)
POST /auth/resend-otp Resend email verification OTP

Requires Authorization: Bearer <access_token>. Silently does nothing if already verified.

Headers
HeaderValue
AuthorizationBearer <access_token>
Response 200
{ "success": true, "message": "OTP sent if applicable" }
POST /auth/forgot-password Send password reset OTP by email

Always returns 200 even if the email doesn't exist — prevents user enumeration.

Request Body (JSON)
FieldTypeRules
email requiredstringValid email address
Response 200
{ "success": true, "message": "If that email exists, a reset OTP has been sent" }
POST /auth/reset-password Reset password with OTP

Validates the OTP from the reset email, sets the new password, and revokes all existing refresh tokens (forces logout everywhere).

Request Body (JSON)
FieldTypeRules
email requiredstringSame email used in forgot-password
otp requiredstring6-digit code from email
new_password requiredstring8–128 chars
Response 200
{ "success": true, "message": "Password reset successful" }
Error codes
StatusMessage
400Invalid or expired reset code (also returned for wrong email — no enumeration)
Token usage in Flutter:
1. Store access_token in memory, refresh_token in secure storage (flutter_secure_storage).
2. Add Authorization: Bearer <access_token> to every authenticated request.
3. On 401, call POST /auth/refresh → save the new token pair → retry the original request.
4. On logout, call POST /auth/logout then clear secure storage.

GET /api/stats/illogical-promo Detect fake / inflated "ancien prix" claims

Returns the most flagrant case where a shop's claimed "regular price" is significantly above the real market floor — exposing misleading discount claims.

Detection Criteria
RuleValue
Min shops selling same product≥ 3
Claimed regular_price vs market floor> 1.6×
Claimed discount≥ 25%
Offender's current price vs market floor≤ 1.15×
Response

  "item": {

    "brand":              "X",

    "image":              "https://...",
    "shopName":           "SomeShop",
    "oldPrice":           45.000,   // inflated "regular price"

    "honestMin":          17.900,   // real market min across all shops

    "effectiveDiscountPct": 3,       // real saving vs market (~3%)
